# 注音挑戰小學堂 - 專案架構文件

本專案是一個基於 **Vite + React + TypeScript** 開發的互動式幼兒注音拖放學習遊戲，結合了後端多維度資料庫設計、Express API 服務、n8n 自動化流程整合，以及圖片與資料的自動化優化腳本。

---

## 3. 核心模組設計規格

### A. 前端遊戲邏輯 (`src/components/ZhuyinDragDrop.tsx`)
- **動態題目載入**：遊戲初始化時，會透過 `/api/quizzes` 向 Express API 請求載入 208 筆注音題目（若載入失敗則 fallback 至 `quizzes_seed.json` 或靜態狀態），實現前後端即時同步。
- **動態介音版面 (2-Cabin / 3-Cabin Mode)**：
  - 前端會自動解析當前字彙的注音，若含有介母（`medial` 不為空），則畫面會自適應調整版面尺寸，拉長顯示 3 個注音格子（聲母、介母、韻母）的垂直佈局。
  - 若無介母，則顯示 2 個注音格子（聲母、韻母）的垂直佈局，並動態放大字卡大小 (`w-36 h-36`) 以利幼兒手指操作。
- **雙向電信與紀錄整合 (Dual-telemetry)**：
  - 當答題成功或失敗時，會同時向本機 Express 服務（發送作答歷程至本地 PostgreSQL）以及外網的 n8n webhook 發送作答封包，確保後台分析與遠端流程同步接收數據。
  - **資料格式**：
    - `action`: `"practice-log"` 或 `"error-analysis"`
    - `quizId`: 識別代號
    - `isCorrect` / `correct`: 答題對錯 (boolean)
    - `spentSeconds`: 作答耗時
    - `subject`: 固定為 `"zhuyin"`
    - `userAnswer` / `user_answer`: 玩家拼湊的卡片組合 `{ initial, medial, final, tone }`
    - `wrongPart` / `wrong_part`: 標識拼錯元件的字串陣列（可包含 `"initial"`, `"medial"`, `"final"`, `"tone"`），答對時為 `[]`。

### B. 圖片自動化優化 (`optimize-images.js`)
- **功能**：自動掃描 `src/assets/raw-images/` 目錄下的所有 `.png`、`.jpg` 圖片。
- **優化邏輯**：使用 `sharp` 等比例縮放至 `512x512` 內，保留透明通道並輸出至 `public/images/`，轉換為品質 `75` 的 WebP，檔名改為小寫。

### C. 208 筆核心中文字資料庫種子腳本 (`server/seed-quiz.js`)
- **一字一圖認知最佳化**：
  - 為了解決「小狗」會先教「小」等幼兒認知偏差，我們精確將所有詞彙重寫並縮減為代表圖片的**「單一核心中文字」**（如「小狗」改為「狗」、「大象」改為「象」、「鱷魚」改為「鱷」、「辣椒」改為「椒」）。
- **注音拆解器 (`decomposeZhuyin`)**：
  - 精確將 208 個核心中文字拆解為 `initial` (聲母), `medial` (介母), `final` (韻母), `tone` (聲調)。若為一聲則標記為 `"1"`，無聲母或無介母時對應欄位留空，與前端 2/3 艙介面邏輯完全契合。
- **資料庫與備份輸出**：
  - 讀取 `.env` 內 `DATABASE_URL`，以資料庫事務清空並重新寫入 PostgreSQL 上的 `ZhuyinQuiz` 扁平化表結構。
  - 同步輸出 `quizzes_seed.json` (前端 JSON 回退備份) 與 `quizzes_seed.sql` (TablePlus 可直接導入的 SQL 種子檔)。

### D. 後端 Express 伺服器 (`server/index.js`)
- **API 連接埠**：預設運行於 `3001` port，透過 Vite proxy 將 `/api` 請求無縫轉發。
- **API 路由**：
  - `GET /api/quizzes`：從 `ZhuyinQuiz` 取得全部題庫。
  - `POST /api/practice-logs`：寫入作答紀錄至 `PracticeLog` 表中。
  - `POST /api/error-analysis`：寫入錯題分析至 `ErrorAnalysis` 表中，如 `wrongAnswer` (玩家答案) 與 `wrongPart` (拼錯的位置)。

### E. 資料庫初始化腳本 (`server/init-db.js`)
- **扁平化 Schema 建立**：讀取 `DATABASE_URL`，於資料庫建立包含 `ZhuyinQuiz` 扁平結構在內的 6 張核心表：
  - `"StudentProfile"`：學生檔案表格
  - `"PracticeSession"`：練習歷程紀錄表
  - `"PracticeLog"`：答題流水誌表格
  - `"ErrorAnalysis"`：深度錯題分析表
  - `"ZhuyinQuiz"`：扁平化注音題庫（欄位包括 `id`, `word`, `image`, `zhuyin`, `initial`, `medial`, `final`, `tone`, `difficulty`, `category`, `tags`）
  - `"EnglishQuiz"`：英文自然拼讀題庫

## 5. 開發與部署常用指令

在專案根目錄下，您可以使用以下快捷指令：

* **啟動前端開發伺服器** (運行在 Port 5173)：
  ```bash
  npm run dev
  ```
* **啟動後端 Express API 伺服器** (運行在 Port 3001)：
  ```bash
  npm run server
  ```
* **一鍵圖片壓縮與優化** (將 raw 圖片輸出成 public webp)：
  ```bash
  npm run optimize
  ```
* **資料庫初始化建表** (在 `DATABASE_URL` 中建置扁平化表格結構)：
  ```bash
  npm run db:init
  ```
* **執行 208 筆核心單字題庫資料庫種子匯入** (並更新 JSON 與 SQL 備份)：
  ```bash
  npm run seed
  ```
* **前端生產環境打包**：
  ```bash
  npm run build
  ```
